Collaboration Built-in with Git Version Control

Static Website Manager is special because it stores all of your website data in a git repository. Git repositories are a type of distributed version control – this makes collaboration easy because you can work on your website independently of other team members via branches, as well as view the project history to see all previous changes.

Production Branch

The production branch is the most important branch. We base it off of your repository’s master branch, but refer to it throughout the application as your “Production Branch.” Whenever the production branch is ahead of another branch (or they’ve diverged), we’ll prompt the user of the other branch to pull in the production changes (git rebase master); if the production branch is behind another branch, we’ll prompt the user of the other branch that they can push their changes to production (git merge master).

Staging and Custom Branches

Everyone in your team gets access to their own staging branch. The internal git branch name will be static_user_ID, where ID is the unique Static Website Manager ID, although throughout the application we’ll refer to these branches as “Your Staging Branch” or “Olivia’s Staging Branch”, depening on the context. In addition to the production and staging branches, you can create custom branches, or “feature branches”, to test out new designs or ideas.

Branch Permissions

Everyone in your team has access to their own staging branch, but you can optionally grant team members access to the production branch, custom branches, or even other team member’s staging branches. All branch’s can pull in production changes, but only those users with production access will be able to push to production.

Blogging and Content Management

Static Website Manager’s CMS is easy to use while maintaining the full power of custom designs and Liquid tags. The Static Website Manager editor has support for both YAML Front-Matter and HTML/Markdown content. Any changes trigger a website build and your updates can be securely and privately previewed. Plus, built-in support for three roles makes it easy to invite collaborators: writers have access to drafts and posts; editors gain access to pages, collections, data and files; and designers additionally have access to layouts and include files.

Deploy your Branches to your own External Accounts

Static Website Manager provides private previews of your compiled website branches through our web-based editor, but Deployments let you link a branch to one or more external accounts. Every change made to a linked branch will trigger a new build of that branch’s website to be pushed to the linked account. The only currently supported deployment is Amazon AWS S3.

Amazon AWS S3

Have an S3 bucket you’d like to deploy to? Just provide your bucket name, the AWS region and IAM credentials that have access to that bucket and we’ll deploy your compiled static website to that bucket any time a change on that branch is made.

Animate your Website with Form Responders

Form Responders provide a unique way for you to capture data from your deployed website. Each Form Responder reveals a unique URL that you can point your HTML forms to, and then attach email addresses and data files!

Send Form Data Emails

Provide up to 10 email addresses and we’ll notify each recipient whenever a new form has been submitted. We’ll include the contents of the form submission in the email so you can react accordingly.

Commit to Data Files

Specify a data file and the form submission will be commited directly to that data file! If the data file is a CSV file or if it’s a JSON or YAML file and contains an array of data, then create forms that POST to the responder’s unique URL and the submitted data will be appended to the data file as a git commit in the branch the form responder was saved in. If the data file is a JSON or YAML file and you POST (or PUT or PATCH) to a path beyond the unique URL, then the key corresponding to the path will be set to the submitted form data (e.g. if the Form Responder URL is “https://www.staticwebsitemanager.com/form-responders/a4de51” and you point your form to “https://www.staticwebsitemanager.com/form-responders/a4de51/some/nested/key” then any submitted data will be set to the key “[a4de51, some, nested]” in the linked data file).
